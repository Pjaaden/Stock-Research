# Investment OS · 操作指南

> 写给没有编程基础的使用者。读完这份指南，你会清楚地知道工作台的每一个功能怎么用、数据从哪来、改了会发生什么。

---

## 系统结构（三分钟理解）

整个系统由三个部分组成，各司其职：

```
GitHub 仓库（数据中枢）
    ├── data/companies.json   ← 六家公司的估值数据、情景、清单
    ├── data/watchlist.json   ← 候选池（待研究公司）
    └── data/logs.json        ← 操作日志存档

工作台 Investment_OS/index.html（展示层）
    ← 从 GitHub 读取数据，渲染成你看到的界面
    → 你的修改经过「同步」按钮写回 GitHub

Claude（研究和更新层）
    ← 分析年报、更新估值框架
    → 直接修改 companies.json 并推送到 GitHub
```

**关键认知**：工作台本身不存储任何数据。它每次打开都从 GitHub 重新读取最新内容。数据的唯一真实来源是 GitHub 上的 JSON 文件。

## 访问工作台

**地址**：https://pjaaden.github.io/Stock-Research/Investment_OS/

在任何浏览器直接打开即可。手机、平板、电脑均可访问，显示同一份数据。

**打开后会发生什么**：
1. 工作台自动去 GitHub 拉取最新的 `companies.json` 和 `watchlist.json`
2. 同时尝试从网络获取实时汇率（USD/CNY、HKD/CNY）
3. 顶部左下角出现「✓ 已同步」绿色标签 → 数据是最新的
4. 出现「⚠ 离线」红色标签 → 网络有问题，显示的是上次缓存的数据

**手动刷新**：点击侧边栏底部「⟳ 刷新·更新汇率」，或直接刷新浏览器页面。

## 界面导航

侧边栏有四个主入口：

| 入口 | 用途 |
|------|------|
| ◉ 组合总览 | 所有持仓的全局视图，金融总资产、三市场资金池、偏离度轨道 |
| ⊞ 公司池 | 管理候选公司、激活/隐藏公司 |
| ≡ 操作日志 | 查看和记录每次决策 |
| ⚙ 设置·同步 | 配置 PAT、输入各市场总资金、手动调整汇率 |

侧边栏「持仓·关注」区域列出所有激活公司，点击任意一家进入**单股详情页**。

## 组合总览页

### 金融总资产（顶部深蓝色区域）

显示三个资金池折算为人民币后的合计总资产。只有在「设置」里输入各市场总资金后才会显示完整数字。

汇率来源标注在右侧：`ExchangeRate-API`（自动获取）或`手动`（使用你在设置里填的数字）。

### 三个资金池卡片

CNY池 / HKD池 / USD池 各自独立展示：
- 已投入金额（以本币计）
- 现金余额
- 投入占比进度条

### 两张图表

左图：三市场持仓的 RMB 等值占比（甜甜圈图）  
右图：每个市场的持仓 vs 现金对比（柱状图）

> 图表是根据你填入的股数和当前价格实时计算的。如果股数未填，图表会显示空白。

### 公司卡片

每张卡片显示：
- 信号标签（↑买入 / →持有 / ↓卖出 / ◎观察）
- 偏离度轨道：蓝色圆点在轨道上的位置代表当前价格相对于熊/牛目标价的位置
- 安全边际百分比（绿色=低估，橙色=轻微高估，红色=明显高估）
- 市价 / 内在价值 / 持股数 / 持仓市值 / 浮盈

点击任意公司卡片进入单股详情。

## 单股详情页（五个标签）

### 标签一：价格梯度

**上半部分：偏离度仪表盘**
大号轨道图，圆点越靠左越低估，越靠右越高估。同时显示：内在价值 / 当前市价 / 距牛市目标% / 距熊市目标%。

「当前价格」输入框：你可以手动修改这里的价格，梯度表和仪表盘会即时更新。这只是本地显示覆盖，**不会自动同步到 GitHub**，需要你点「↑ 同步此公司」才会保存。

**下半部分：价格行动梯度表**
六个区间，当前价格所在区间会高亮显示（蓝色背景），并在表格上方的状态栏显示对应操作指令。

| 区间 | 操作指令 |
|------|---------|
| 极度低估 | BUY（积极买入至满仓） |
| 合理低估 | ADD（正常建仓） |
| 公允偏低 | HOLD（持有，等催化剂） |
| 公允偏高 | HOLD（持有，不加仓） |
| 轻微高估 | TRIM（减仓50%） |
| 明显高估 | SELL（止盈清仓） |

### 标签二：估值情景

三情景表格：牛市 / 基准 / 熊市，每个情景显示概率、目标价、假设条件、当前价与目标价的偏离百分比、对加权价值的贡献。

底部「加权内在价值」= 三个目标价按概率加权的均值。安全边际 = (加权内在价值 - 当前价) / 加权内在价值。

**这一页的数字来自 companies.json，由 Claude 在研究更新时修改。你不需要在这里手动改任何东西。**

### 标签三：仓位管理

**这里是你最常用的输入区域。**

| 字段 | 说明 |
|------|------|
| 持有股数 | 填入你实际持有的股票数量（单位：股，不是手/万） |
| 平均成本 | 买入均价，用于计算浮盈 |
| 最大仓位上限 | 这只股票在你总资金中的上限比例（%），用于梯度表显示目标仓位 |

填完后点**「↑ 同步此公司至 GitHub」**才会保存到云端。下次在任何设备打开工作台都能看到你填的数据。

下方还有系统仓位规则表，来自 companies.json 的研究数据，供参考。

### 标签四：监控清单

来自研究报告的核心核查项，每项都有：
- 触发行动（核查后应该做什么）
- 优先级（高/中/低）
- 分类（年报验证 / 外部因素 / 产能扩张等）

点击左侧的小方框可以打勾/取消，点「↑ 同步勾选」保存到 GitHub。

**季报或年报发布后**，根据实际数据逐项核查，打勾已验证的项目。如果发现触发行动，告知 Claude 更新对应的估值参数。

### 标签五：研究笔记

显示 Claude 整合的完整研究结论，包括核心利润结构、关键财务数据、方法论说明、战略判断等。**只读，不可编辑。** 需要更新时告知 Claude 修改 companies.json。

## 公司池页面

分三个区：

**自选·持仓（已激活）**：当前在总览页显示的公司。点「详情」进入单股详情页，点「隐藏」从总览页移除（数据保留，随时可恢复）。

**候选池**：来自 `watchlist.json` 的轻量条目，只有公司名、代码、关注理由、优先级，没有估值数据。可以升级为「研究中」状态。你可以直接在工作台添加新候选（侧边栏「＋ 加入候选池」），但需要点「↑ 同步至 GitHub」才会保存。

**已隐藏**：点「展开▾」查看，可以恢复任何被隐藏的公司。

## 设置页面

### GitHub 同步配置

**PAT Token**：你的 GitHub 个人访问令牌，相当于工作台写入 GitHub 的钥匙。只需配置一次，保存在浏览器里，不会发送到任何其他地方。

点「测试连接」确认 PAT 有效后，所有「↑ 同步」按钮才能正常工作。

**如果换了浏览器或换了设备**，需要重新在设置里填入 PAT。这是唯一需要重新配置的东西。

### 汇率设置

工作台启动时会自动从两个外部服务获取实时汇率。如果网络受限无法获取，会使用这里的手动值。可以点「⟳ 重新获取实时汇率」手动触发更新。

### 三个资金池

分别填入你在 A股 / 港股 / 美股账户的总资金（以各自货币的万元为单位）。这决定了总览页的资金池进度条和现金余额显示。**这些数字只存在你的浏览器里，不会推送到 GitHub。**

## 操作日志

每次执行交易前，先在这里记录决策依据：
- 选择公司和操作类型（买入/加仓/减仓/清仓/持有确认）
- 填入执行价格和股数变动
- 写明依据：「当前安全边际+20%，触发合理低估区间的ADD指令」

日志分两类：
- 来自 `data/logs.json` 的历史记录（由 Claude 存入）
- 你在工作台新建的本地记录（标注「本地」标签）

点「↓ 导出」可以下载完整日志为 JSON 文件，发给 Claude 后可以合并到 logs.json 永久存档。

## 日常使用流程

### 每天（2分钟）

1. 打开工作台，确认左下角显示「✓ 已同步」
2. 看组合总览，注意有无信号变化（偏离度轨道位置、安全边际颜色）
3. 如果当天有交易，在操作日志记录依据

### 季报/年报发布后（10–30分钟）

1. 打开对应公司的单股详情页 → 监控清单
2. 根据实际数据逐项核查，打勾已验证的项目
3. 将核查结果告知 Claude（例如：「赛轮Q1季报：EPS 0.22元，越南利润率维持22%，新和平亏损收窄至-3亿，第2、3条监控打勾」）
4. Claude 更新 companies.json 并推送，刷新工作台即见新数据

### 重大判断变化时（告知 Claude）

直接说明要改什么，越具体越好：

> 「赛轮：伊朗停火协议达成，油价回落至$82。把牛市情景概率从25%调到40%，基准从50%降到45%，熊市从25%降到15%。牛市目标价从14提到16。」

Claude 修改并推送后，刷新工作台即可看到更新后的估值和信号。

## 两个规则

**规则一：高频数据你自己改，低频数据告诉 Claude**

| 你自己在工作台改 | 告知 Claude 更新 |
|-----------------|----------------|
| 持有股数 | 估值情景（目标价、概率） |
| 平均成本 | 监控清单文字 |
| 当前价格（覆盖） | 研究笔记 |
| 监控清单勾选 | 仓位规则 |
| 候选池条目 | 候选公司的基本信息 |

**规则二：改完一定要点「↑ 同步」**

不点同步，数据只在浏览器里。关掉标签页就消失了，换设备也看不到。点了同步，数据写进 GitHub，永久保留，任何设备都能看到。

## 常见问题

**Q：换了电脑，为什么看不到我填的股数？**  
A：很可能同步时 PAT 未配置，数据没写进 GitHub。在新设备的设置页填入 PAT，然后去仓位管理重新填入股数并同步。

**Q：工作台显示的是旧数据，刷新也没用？**  
A：点侧边栏底部「⟳ 刷新·更新汇率」，或等待几分钟后再刷新（GitHub Pages 有缓存延迟，通常不超过5分钟）。

**Q：「↑ 同步」按钮点了没反应或报错？**  
A：PAT 可能过期或权限不够。进设置页点「测试连接」，如果失败需要去 GitHub 重新生成 PAT（Settings → Developer settings → Personal access tokens，权限选 Contents: Read and write）。

**Q：想给一家公司加一条监控清单，怎么办？**  
A：告知 Claude 即可。说清楚公司名、要加的监控内容、触发行动、优先级，Claude 会更新 companies.json 并推送。

**Q：想把一家公司从工作台移除但保留研究报告？**  
A：进公司池页面，点该公司的「隐藏」按钮，再点「↑ 同步至 GitHub」。公司从总览消失，研究 HTML 文件不受影响。
